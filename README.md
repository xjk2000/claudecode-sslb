## 核心流程

### 兵部 TDD 流程
```
RED (写测试) → 失败 → 打回五监 → GREEN (验证) → 通过
```

兵部只负责测试验证，测试失败后打回给五监修复。

### 刑部调试流程
```
收集症状 → 定位根因 → 分析归责 → 打回五监修复
```

刑部只负责调试诊断发现问题，分析完成后打回给五监修复。

### 工部审查流程
```
审查代码 → 发现问题 → 反馈五监 → 确认修复 → 批准合并
```

## 三省职责

### 中书省（决策中枢）
- **中书令**: 首席秘书，接收用户指令
- **中书舍人**: 记录秘书，分发敕令

### 门下省（审议封驳）
- **侍中**: 首席审核官，决定封驳
- **给事中**: 细节审核官，技术审查

### 尚书省（执行中枢）
- **尚书令**: 首席调度官，协调执行

## 使用说明

1. 克隆仓库到 Claude Code 项目目录
2. 根据任务类型选择对应的 Agent 和 Skill
3. 遵循敕令模板进行任务流转
4. 通过打回机制确保五监修复问题
