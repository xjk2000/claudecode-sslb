## Phase 1 — 委派 @中书舍人 执行项目全局调研

中书舍人需产出 **4 份分层文档**，存入 `我的帝国朝堂/弘文馆/`：

### 文档 1：项目全貌（`弘文馆/项目全貌.md`）

调研范围：
- **项目结构**：目录布局、模块划分、入口文件
- **技术栈**：语言版本、框架、构建工具、依赖管理
- **架构模式**：分层架构/DDD/微服务/前后端分离等
- **代码规模**：文件数量、代码行数、测试覆盖率概况
- **关键配置**：数据库、缓存、消息队列、外部服务等
- **开发规范**：lint 规则、代码风格、CI/CD 配置
- **近期动态**：最近 git 提交趋势、活跃开发区域
- **现有文档**：README、ARCHITECTURE、CHANGELOG 等
- **风险与技术债**：潜在问题、遗留代码、依赖过期等

### 文档 2：模块关系图（`弘文馆/模块关系图.md`）

调研范围：
- **前端模块**：页面/路由 → 组件 → 状态管理 → API 调用层
- **后端模块**：Controller → Service → Repository/Mapper → Entity/Model
- **模块间依赖**：谁调谁、共享模块、公共工具类
- **外部服务依赖**：第三方 API、微服务间调用、消息队列生产/消费
- **用 Mermaid/文本关系图表达模块间的调用关系**

### 文档 3：API 接口清单（`弘文馆/API接口清单.md`）

调研范围：
- **后端 API**：逐个列出接口路径、方法、对应 Controller、入参、出参
- **前端调用映射**：每个 API 被哪些前端页面/组件调用
- **接口分组**：按业务模块分组（如用户模块、订单模块等）
- **认证/权限**：接口的权限要求
- 如果项目有 Swagger/OpenAPI，优先从中提取

### 文档 4：数据模型总结（`弘文馆/数据模型总结.md`）

调研范围：
- **数据库表/集合**：表名、核心字段、表间关系（外键/关联）
- **实体/Model 类**：与表的映射关系
- **DTO/VO 对象**：接口层数据转换对象
- **用文本 ER 图或表格表达核心表之间的关系**
