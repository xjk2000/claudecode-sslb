# 文档管理技能
礼部（Ritual Ministry）执行的标准文档管理流程。

## 核心职责

- 文档规划与结构设计
- 技术文档编写
- 文档质量审核
- 文档版本管理
- 知识库维护

## 文档编写流程

### Phase 1: 规划
1. 确定文档目标读者
2. 明确文档范围和边界
3. 设计文档结构
4. 制定编写计划

### Phase 2: 编写
1. 遵循文档模板
2. 使用简洁清晰的语言
3. 包含足够的示例
4. 引用相关资源

### Phase 3: 审核
1. 检查内容准确性
2. 验证代码示例
3. 检查格式一致性
4. 获取相关人员审核

### Phase 4: 发布
1. 合并到文档仓库
2. 更新索引和导航
3. 通知相关人员
4. 收集反馈改进

## 文档质量检查清单

### 内容质量
- [ ] 内容是否准确无误？
- [ ] 是否覆盖所有要点？
- [ ] 是否有遗漏的重要信息？
- [ ] 逻辑是否清晰连贯？

### 可读性
- [ ] 语言是否简洁明了？
- [ ] 术语使用是否一致？
- [ ] 格式是否统一规范？
- [ ] 段落长度是否适中？

### 示例质量
- [ ] 代码示例是否可运行？
- [ ] 示例是否足够完整？
- [ ] 边界情况是否覆盖？
- [ ] 注释是否清晰？

### 可维护性
- [ ] 是否有更新日期标记？
- [ ] 文档结构是否易于扩展？
- [ ] 图片和附件是否正确链接？

## 协作编写规范

### 提交规范
```bash
# 文档提交格式
git commit -m "docs: [文档类型] [简短描述]"
git commit -m "docs: README 添加安装说明"
git commit -m "docs: API 添加用户管理接口"
git commit -m "fix: 修正拼写错误"
```

### Review 规范
1. 文档变更需要至少 1 人审核
2. 敏感内容需要 2 人审核
3. 审核时间不超过 48 小时
4. 审核意见需在 7 天内处理
