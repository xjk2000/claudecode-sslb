# 数据处理技能
户部（Treasury Ministry）执行的标准数据处理和分析流程。

## 核心职责

- 数据收集与验证
- 数据转换与处理
- 数据分析与洞察
- 数据可视化呈现
- 数据质量保障

## 数据处理流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   COLLECT   │───►│   CLEANSE   │───►│   ANALYZE   │───►│   PRESENT   │
│   收集      │    │   清洗      │    │   分析      │    │   呈现      │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### Phase 1: 数据收集
1. 识别数据来源
2. 验证数据可用性
3. 获取必要访问权限
4. 记录数据 schema

### Phase 2: 数据清洗
1. 处理缺失值
2. 识别和处理异常值
3. 数据类型转换
4. 标准化和归一化

### Phase 3: 数据分析
1. 描述性统计
2. 趋势分析
3. 相关性分析
4. 模式识别

### Phase 4: 数据呈现
1. 选择合适的可视化
2. 制作图表
3. 编写分析报告
4. 提出建议

## 数据质量检查清单

### 完整性
- [ ] 数据是否有缺失？
- [ ] 缺失比例是否可接受？
- [ ] 是否有填充策略？

### 一致性
- [ ] 数据格式是否统一？
- [ ] 命名规范是否一致？
- [ ] 时间格式是否标准？

### 准确性
- [ ] 数据是否在合理范围内？
- [ ] 是否有明显错误？
- [ ] 是否与已知事实矛盾？

### 时效性
- [ ] 数据是否是最新的？
- [ ] 是否需要定期刷新？
- [ ] 历史数据是否可追溯？

## 数据操作指南

### 数据转换
```markdown
## 常用数据转换

### 数值转换
- 标准化: (x - mean) / std
- 归一化: (x - min) / (max - min)
- 对数变换: log(x)
- 分箱: 连续值转为离散区间

### 文本转换
- 小写化: text.lower()
- 分词: text.split()
- 词干提取: Porter/Stemmer
- 停用词移除

### 日期转换
- 解析: parse(date_string)
- 提取: year, month, day, weekday
- 差值: (date2 - date1).days
```

### 数据聚合
```markdown
## 聚合操作

### 分组聚合
- 按类别求和
- 按时间窗口汇总
- 多维度交叉分析

### 统计指标
- 总和 (SUM)
- 平均值 (AVG)
- 中位数 (MEDIAN)
- 标准差 (STD)
- 计数 (COUNT)
- 去重计数 (COUNT DISTINCT)

### 排名和分位
- TOP N / BOTTOM N
- 分位数 (QUARTILE)
- 百分位 (PERCENTILE)
```

## 数据可视化原则

### 选择合适图表

| 数据类型 | 推荐图表 |
|----------|----------|
| 趋势 | 折线图 |
| 比较 | 柱状图 |
| 构成 | 饼图/堆叠图 |
| 分布 | 直方图/箱线图 |
| 关系 | 散点图 |
| 地理 | 地图 |

### 设计规范
1. **清晰标题**: 一眼看懂图表主题
2. **适当标注**: 坐标轴、数据点标注
3. **图例说明**: 区分不同数据系列
4. **颜色编码**: 合理使用颜色传达信息
5. **比例协调**: 避免截断或误导
