# 尚书省任务执行技能

## 概述

执行实现计划，批量执行并设置检查点。

**核心原则**: 每次任务一小步，检查点验证。

## 何时使用

当有实现计划时：
- 任务相互独立
- 适合在同一会话执行
- 需要分批检查点

## 任务分解

### 分解原则
- 每个任务 2-5 分钟
- 有明确的文件和步骤
- 有验证命令
- 独立可测试

### 分解模板

```markdown
## 任务分解

### 任务N: [组件名]
**负责**: [五监部门]
**文件**:
- 创建: `exact/path/to/file.py`
- 修改: `exact/path/to/existing.py:123-145`

**步骤**:
1. [ ] 步骤1
2. [ ] 步骤2
3. [ ] 步骤3

**验证**:
```bash
pytest tests/path/test.py -v
```
```

## 五监分派

### 分派规则

| 任务类型 | 分派部门 |
|----------|----------|
| 核心业务逻辑 | 将作监 |
| 前端UI/交互 | 少府监 |
| 安全相关 | 军器监 |
| 数据处理 | 都水监 |
| 框架/架构 | 国子监 |
| 测试 | 兵部 |
| 调试 | 刑部 |
| 审查 | 工部 |

### 分派敕令模板

```markdown

---
尚书令（签字）: ___________
```

## 检查点机制

### 检查点类型
1. **任务完成检查**: 每个任务完成后检查
2. **批次检查**: 每3个任务后检查
3. **阶段检查**: 每个功能模块完成后检查
