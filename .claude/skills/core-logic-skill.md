# 核心业务逻辑技能
将作监（Directorate of Construction）执行的核心逻辑开发流程。

**核心原则**：将作监只负责实现和修复，发现问题后按流程处理。

## 核心逻辑开发流程

### Phase 1: 需求分析
1. 理解业务背景和目标
2. 识别业务规则和约束
3. 定义业务术语
4. 明确输入输出

### Phase 2: 逻辑设计
1. 绘制业务流程图
2. 设计状态机（如果适用）
3. 定义数据结构
4. 规划算法复杂度

### Phase 3: 代码实现
1. 遵循 SOLID 原则
2. 编写清晰注释
3. 添加边界检查
4. 错误处理完善

### Phase 4: 自测验证
1. 编写单元测试
2. 边界条件测试
3. 验证通过后回复敕令

## 领域模型设计

### 设计原则
```markdown
## 领域模型设计指南

### 核心概念
1. **实体 (Entity)**: 有唯一标识的对象
2. **值对象 (Value Object)**: 无唯一标识的不可变对象
3. **聚合 (Aggregate)**: 一组相关对象的集合
4. **领域服务 (Domain Service)**: 不属于任何实体的业务操作

### 设计检查
- [ ] 实体标识是否清晰？
- [ ] 值对象是否不可变？
- [ ] 聚合边界是否合理？
- [ ] 领域逻辑是否内聚？
```

## 算法设计

### 算法选择指南
```markdown
## 复杂度选择原则

| 数据规模 | 推荐复杂度 | 适用算法 |
|----------|------------|----------|
| n ≤ 10 | O(n!) | 全排列 |
| n ≤ 20 | O(2^n) | 子集 |
| n ≤ 100 | O(n²) | 简单排序 |
| n ≤ 10^5 | O(n log n) | 高效排序 |
| n ≤ 10^7 | O(n) | 线性扫描 |
```

### 常见算法模式
- **双指针**: 有序数组、链表
- **滑动窗口**: 子数组/子串问题
- **动态规划**: 最优子结构
- **回溯**: 穷举搜索
- **分治**: 归并排序
