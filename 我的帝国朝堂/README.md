# 皇帝行宫 — 三省六部过程记录

此目录是三省六部 Agent 协作系统的**持久化记录中心**，所有 Agent 在执行过程中必须在此留下记录。

> **此目录位于项目根目录下**（`<项目>/我的帝国朝堂/`），每个项目有自己独立的过程记录。
> 首次使用三省六部时，**SessionStart hook 会自动创建**此目录结构。

## 目录结构

```
<项目根目录>/我的帝国朝堂/
│
│  ── 三省中枢 ──
├── 政事堂/              当前活跃敕令（诏-YYYYMMDD-XXX/ 文件夹式存储）
├── 秘书省/              归档敕令文件夹
├── 弘文馆/              敕令总结库（中书舍人撰写）
├── TEMPLATE-edict.md    敕令模板
│
│  ── 六部专属文档库 ──
├── 考功司/              吏部（Agent能力评估、任务分配）
├── 籍账库/              户部（数据分析报告）
├── 职方司/              兵部（测试报告、打回记录）
├── 都官司/              刑部（调试报告、根因分析）
├── 礼部精舍/            礼部（技术文档、API文档）
├── 营缮司/              工部（代码审查报告）
│
│  ── 五监专属文档库 ──
├── 将作图谱库/          将作监（核心业务实现记录）
├── 百工署/              少府监（前端组件实现记录）
├── 甲弩坊署/            军器监（安全实现记录）
├── 河渠书库/            都水监（数据库变更记录）
└── 经籍库/              国子监（架构决策记录）
```

## 政事堂

**当前的工作台。** 所有进行中的敕令都存放在这里。

- 每个敕令一个文件夹：`诏-YYYYMMDD-XXX/`，内含敕令主文件、执行计划、打回记录等
- 中书令创建敕令后，中书舍人创建敕令文件夹并写入政事堂
- 尚书令执行计划、五监实现进度、打回记录等全部存入敕令文件夹
- 侍中验收通过后，中书舍人将整个敕令文件夹移入秘书省，并撰写总结存入弘文馆

## 秘书省

**历史的档案馆。** 所有已完成敕令的原件归档在这里。

- 侍中验收通过后，中书舍人将敕令原件（含完整执行记录）从政事堂移入
- 被永久废弃的敕令也归档至此（标记为废弃）
- 需要查看敕令详情时可来此查阅

## 弘文馆

**三省六部的知识中枢。** 包含两类文档：

### 基础记忆文档（/init-dynasty 生成，敕令归档时增量更新）
- `项目全貌.md` — 项目宏观认知（技术栈、架构、规模、风险）
- `模块关系图.md` — 前后端模块依赖关系、调用链路
- `API接口清单.md` — 接口路径、前后端对应关系
- `数据模型总结.md` — 数据库表、实体映射、表关系

### 敕令总结（每次敕令完成后生成）
- `ZS-YYYYMMDD-XXX-summary.md` — 任务概述、关键决策、经验教训

**查阅优先级**：基础记忆（快速定位改动点）> 敕令总结（历史经验）> 秘书省原件（详细记录）

## 六部文档库

**每个部门任务完成后，必须写入任务记录到自己的文档库。** 文件命名：`ZS-YYYYMMDD-XXX-[部门名].md`

| 部门 | 文档库 | 目录 | 存放内容 |
|------|--------|------|----------|
| 吏部 | 考功司 | `考功司/` | 能力评估报告、任务分配建议 |
| 户部 | 籍账库 | `籍账库/` | 数据分析报告、数据质量评估 |
| 兵部 | 职方司 | `职方司/` | 测试清单、打回记录、覆盖率变化 |
| 刑部 | 都官司 | `都官司/` | 根因分析、调试过程、验证结果 |
| 礼部 | 礼部精舍 | `礼部精舍/` | 技术文档产出清单、审核结果 |
| 工部 | 营缮司 | `营缮司/` | 审查结论、代码质量评价、打回记录 |

## 五监文档库

**每个五监任务完成后，必须写入任务记录到自己的文档库。** 文件命名：`ZS-YYYYMMDD-XXX-[监名].md`

| 部门 | 文档库 | 目录 | 存放内容 |
|------|--------|------|----------|
| 将作监 | 将作图谱库 | `将作图谱库/` | 核心业务实现、设计决策、打回修复 |
| 少府监 | 百工署 | `百工署/` | 前端页面/组件清单、API 调用映射 |
| 军器监 | 甲弩坊署 | `甲弩坊署/` | 安全变更清单、风险评估 |
| 都水监 | 河渠书库 | `河渠书库/` | 数据库变更、迁移脚本、实体映射 |
| 国子监 | 经籍库 | `经籍库/` | 架构决策（ADR）、配置变更 |

## 为什么需要这个目录

1. **上下文锚定** — Agent 读取政事堂可立即恢复工作状态，防止治理漂移
2. **过程可追溯** — 每个决策、审议、打回都有记录
3. **跨会话延续** — 新会话读取政事堂即可继续之前的工作
4. **知识积累** — 弘文馆总结和各部门文档库帮助 Agent 快速获取经验
5. **部门隔离** — 每个部门有专属文档库，职责清晰，避免混乱
6. **检索分层** — 弘文馆总结（快）→ 部门文档（专）→ 秘书省原件（全），从摘要到详情
