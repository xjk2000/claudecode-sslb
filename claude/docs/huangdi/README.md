# 皇帝行宫 — 三省六部过程记录

此目录是三省六部 Agent 协作系统的**持久化记录中心**，所有 Agent 在执行过程中必须在此留下记录。

> **此目录位于项目根目录下**（`<项目>/docs/huangdi/`），每个项目有自己独立的过程记录。
> 首次使用三省六部时，中书令会自动调用 `edict-manager.sh init` 创建此目录结构。

## 政事堂（zhengshitang）

**当前的工作台。** 所有进行中的敕令都存放在这里。

- 中书令创建敕令后，中书舍人将其写入政事堂
- 每个敕令一个文件，包含完整的执行过程记录
- 尚书令执行计划、五监实现进度、打回记录等全部追加到敕令文件中
- 侍中验收通过后，中书舍人将敕令原件移入秘书省，并撰写总结存入弘文馆

## 秘书省（mishusheng）

**历史的档案馆。** 所有已完成敕令的原件归档在这里。

- 侍中验收通过后，中书舍人将敕令原件（含完整执行记录）从政事堂移入
- 被永久废弃的敕令也归档至此（标记为废弃）
- 需要查看敕令详情时可来此查阅

## 弘文馆（hongwenguan）

**知识的总结库。** 每个敕令完成后，中书舍人撰写总结存入此处。

- 存放中书舍人对每次敕令的**总结文档**（非原件）
- 总结包含：任务概述、关键决策、修改文件、经验教训、关键词
- 皇帝询问时**优先查阅此处**，快速定位相关经验

## 为什么需要这个目录

1. **上下文锚定** — Agent 读取政事堂可立即恢复工作状态，防止治理漂移
2. **过程可追溯** — 每个决策、审议、打回都有记录
3. **跨会话延续** — 新会话读取政事堂即可继续之前的工作
4. **知识积累** — 弘文馆总结和各部门文档库帮助 Agent 快速获取经验
5. **部门隔离** — 每个部门有专属文档库，职责清晰，避免混乱
6. **检索分层** — 弘文馆总结（快）→ 部门文档（专）→ 秘书省原件（全），从摘要到详情
