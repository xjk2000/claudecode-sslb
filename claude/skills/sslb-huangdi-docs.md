# 皇帝行宫 — 过程记录规范

## 核心原则

1. **每个敕令必须有记录** — 没有文档的敕令不存在
2. **先读后写** — 每次行动前先读取政事堂，恢复上下文
3. **追加不覆盖** — 过程记录只追加，不删除历史内容
4. **完成即归档** — 验收通过的敕令原件移入秘书省，中书舍人写总结存入弘文馆

## 重要说明

`docs/huangdi/` 是**项目级**目录，位于当前项目根目录下（不是 `~/.claude/` 下）。每个项目有自己独立的过程记录。

首次在项目中使用三省六部时，中书令的 Phase 0 会自动调用 `edict-manager.sh init` 创建完整目录结构。

## 各 Agent 读写职责

### 会话开始时（所有 Agent）

**必须**先检查政事堂是否有活跃敕令：
```
读取 docs/huangdi/zhengshitang/ 目录
  → 有活跃敕令 → 读取敕令内容，恢复上下文，继续执行
  → 无活跃敕令 → 正常开始新任务
```

### 中书令 — 创建敕令记录

触发时机：brainstorming 完成、敕令拆解完成后

写入内容：
- 复制 `TEMPLATE-edict.md` → `zhengshitang/ZS-YYYYMMDD-XXX.md`
- 填写：圣旨原文、需求探索记录、敕令内容
- 设置状态为 `📝起草`

### 中书舍人 — 编号登记、状态更新、归档总结、知识检索

**触发时机一**：敕令文档化后、状态变更时

写入内容：
- 确认敕令编号唯一性
- 更新敕令状态（📝起草 → ⏳待审议）
- 在敕令文件中追加分发记录

**触发时机二**：敕令验收通过后

归档操作：
1. 将敕令原件从 `zhengshitang/` 移入 `mishusheng/`
2. 撰写敕令总结存入 `hongwenguan/ZS-YYYYMMDD-XXX-summary.md`

**触发时机三**：皇帝提问时

知识检索顺序：
1. 查弘文馆（总结，快速定位）
2. 查各部门文档库（专属文档）
3. 查秘书省（原始敕令详情）

### 侍中 — 审议记录、验收记录

触发时机：审议敕令时、验收任务时

**审议时写入**：
- 更新「四、门下省审议记录」章节
- 更新状态（⏳待审议 → ✅已放行 / 封驳则追加封驳记录）

**验收时写入**：
- 更新「七、验收记录」章节
- 验收通过 → 状态改为 🟢已完成，**通知中书舍人：① 原件移入秘书省 ② 写总结存弘文馆**
- 验收不通过 → 追加不通过原因，状态保持 🔵执行中

### 尚书令 — 执行计划、进度追踪

触发时机：制定计划时、任务分发时

写入内容：
- 更新「五、执行计划」章节（参与Agent、任务分解）
- 更新状态（✅已放行 → 🔵执行中）
- 计划文件保存至 `docs/huangdi/zhengshitang/ZS-YYYYMMDD-XXX-plan.md`（可选）

### 兵部/刑部/工部 — 打回记录

触发时机：生成打回敕令时

写入内容：
- 在敕令文件「六、执行过程记录」中追加打回记录
- 记录打回编号、类型、原因、打回给谁

### 五监 — 实现记录

触发时机：完成实现/修复后

写入内容：
- 在敕令文件「六、执行过程记录」中追加实现记录
- 记录修改了哪些文件、通过了哪些测试

## 敕令生命周期

```
📝起草（中书令创建）
  → ⏳待审议（提交门下省）
  → ✅已放行（侍中放行）/ 封驳（打回中书令修改）
  → 🔵执行中（尚书令开始执行）
  → 🟢已完成（侍中验收通过）→ 原件移入秘书省 + 总结存入弘文馆
```

## 文件命名规则

| 类型 | 文件名 | 存放位置 |
|------|--------|----------|
| 正式敕令（原件） | `ZS-YYYYMMDD-XXX.md` | zhengshitang/ → mishusheng/ |
| 执行计划（原件） | `ZS-YYYYMMDD-XXX-plan.md` | zhengshitang/ → mishusheng/ |
| 敕令总结 | `ZS-YYYYMMDD-XXX-summary.md` | hongwenguan/ |
| 兵部打回 | 追加到原敕令文件 | — |
| 刑部打回 | 追加到原敕令文件 | — |
| 工部打回 | 追加到原敕令文件 | — |

## 上下文恢复（防漂移）

每次收到用户新指令时，**必须先执行**：

1. 列出 `docs/huangdi/zhengshitang/` 中的文件
2. 如果有活跃敕令，读取最新的敕令文件
3. 根据敕令状态决定：
   - 📝起草 → 继续 brainstorming 或提交审议
   - ⏳待审议 → 等待/执行审议
   - ✅已放行 → 开始/继续执行计划
   - 🔵执行中 → 继续执行，读取进度
4. 确认自己的 Agent 角色，按三省六部流程行动

**这是防止治理漂移的核心机制。**
